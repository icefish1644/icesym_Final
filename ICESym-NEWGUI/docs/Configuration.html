Simulation Configuration

You can access to Simulation Configuration on the first tab of the GUI.

In this section you must select the global parameters to the Simulation, for example, you can choose the frequency to save data (and what data you need) for Post Process Tab, or essential parameters for the simulation, e.g the list of RPM(s) to simulate, or the number of strokes per cycle of the engine.

List of Attributes:

  • Number of RPMS: Number of RPM(s) to simulate, inserted in the table below.

  • Number of Cycles per RPM: Number of cycles for each RPM simulated.

  • Max. Delta Crank Angle: Maximum allowable crank angle variation in each time step [deg].

  • Courant: Courant Number.

  • Ga: Specific heat ratio.

  • Viscous Flow: Switches on/off (1/0) the friction computation with the pipe wall.

  • Heat Flow: Switches on/off (1/0) the heat transfer computation with the pipe wall.

  • R Gas: Gas constant.

  • Number of Strokes: Number of strokes per cycle.

  • Engine Geometry: Engine type:

    • Alternative
    • Opposed-Piston

  • Ignition Order: Ignition order of cylinders. Example: 0,1,3,2,4

  • File Save State: File name to save the state in each time step (specified for iterations per save).

  • File Save Species: File name to save the chemical species concentration in each time step (specified for iterations per append).

  • Get State From: Select how you initialize the state:

    • From Here-> You must indicates the State in each component.
    • From File-> You must indicates the file to load the state for all components. (Used when you need to continue a simulation).
    • From Solver-> ICESym-1D calculates the initial state numerically with a predictor.

  • Find: If you select From File in the previous parameter, you must specified the file that contents the initial state.

  • Saving Frecuency: Step of iterations to write the simulation state (0 is never). The state is saved when number_iteration%ips == 0.

  • Frecuency: Step of iterations to save the component data. In order to see correctly the results in the Post Process you should decrease this value. For higher speed, you should increase this value. (0 is never).

  • Folder Name: Folder name to save the simulation.

  • Calculate Engine Extras: Calculate engine's data to Post Process, such as Power, Torque etc.
